# Обновление: Архивирование новостей и горизонтальная прокрутка

## Новые возможности

### 1. Архивирование новостей

Теперь новости не удаляются навсегда, а могут быть отправлены в архив. Архивные новости:
- Скрыты на публичном сайте
- Видны в админ-панели с пометкой "АРХИВ"
- Могут быть восстановлены в любой момент

#### Как использовать:

**В админ-панели:**
1. Откройте раздел "Новости"
2. Найдите нужную новость
3. Нажмите кнопку "В архив" - новость будет скрыта на сайте
4. Для восстановления нажмите "Восстановить"

**При редактировании:**
- Установите галочку "Архивная (скрыта на сайте)" чтобы отправить новость в архив
- Снимите галочку чтобы вернуть новость на сайт

**Преимущества:**
- История новостей сохраняется
- Можно временно скрыть устаревшие новости
- Легко восстановить при необходимости
- Полное удаление доступно только через кнопку "Удалить"

### 2. Горизонтальная прокрутка новостей

Новости теперь отображаются в современном горизонтальном слайдере:

#### Особенности:
- **Элегантный дизайн** - новости прокручиваются горизонтально
- **Кнопки навигации** - стрелки влево/вправо для удобной прокрутки
- **Плавная анимация** - smooth scrolling для приятного UX
- **Touch-поддержка** - можно листать пальцем на мобильных устройствах
- **Drag & Drop** - можно перетаскивать мышкой на десктопе
- **Адаптивность** - отлично работает на всех устройствах

#### Как это выглядит:
```
[←] [Новость 1] [Новость 2] [Новость 3] [Новость 4] [→]
```

#### Управление:
- **Кнопки ← →** - прокрутка на одну новость
- **Мышь** - зажмите и перетащите для прокрутки
- **Touch** - свайп пальцем на мобильных
- **Колесико мыши** - горизонтальная прокрутка (если поддерживается)

### 3. Технические детали

#### Изменения в API:

**GET /api/content**
- Теперь фильтрует архивные новости (archived: true)
- Возвращает только активные новости для публичного сайта

**PUT /api/admin/news/:id**
- Новый параметр: `archived` (boolean)
- Поддерживает частичное обновление (только archived)

**Структура новости:**
```json
{
  "id": "1234567890",
  "title": "Заголовок",
  "text": "Текст новости",
  "imageUrl": "/images/baner/image.jpg",
  "eventDate": "2024-01-15",
  "order": 1,
  "archived": false,
  "createdAt": "2024-01-15T10:00:00Z"
}
```

#### Новые CSS классы:

**Для слайдера:**
- `.news-slider-container` - контейнер слайдера
- `.news-slider` - область прокрутки
- `.news-slider-track` - трек с новостями
- `.news-slider-item` - отдельная новость
- `.news-slider-nav` - кнопки навигации

**Для админки:**
- `.archived-item` - стиль архивной новости
- `.archive-badge` - бейдж "АРХИВ"
- `.archive-btn` - кнопка "В архив"
- `.restore-btn` - кнопка "Восстановить"

### 4. Совместимость

- ✅ Все современные браузеры (Chrome, Firefox, Safari, Edge)
- ✅ Мобильные устройства (iOS, Android)
- ✅ Планшеты
- ✅ Touch-устройства
- ✅ Старые браузеры (с fallback на обычную прокрутку)

### 5. Производительность

- **Lazy loading** - изображения загружаются по мере необходимости
- **Smooth scrolling** - плавная анимация без рывков
- **Оптимизированный рендеринг** - минимальная нагрузка на браузер
- **Кэширование** - изображения кэшируются браузером

## Миграция существующих данных

Существующие новости автоматически получат поле `archived: false` при первом обновлении.
Никаких дополнительных действий не требуется.

## Рекомендации

1. **Архивируйте старые новости** вместо удаления - это сохранит историю
2. **Используйте качественные изображения** для лучшего вида в слайдере
3. **Оптимальное количество новостей** - 5-10 активных новостей
4. **Порядок отображения** - используйте поле "Порядок" для управления последовательностью

## Поддержка

При возникновении проблем проверьте:
- Версию браузера (рекомендуется последняя)
- JavaScript включен
- Нет ошибок в консоли браузера (F12)
- Сервер запущен и доступен
